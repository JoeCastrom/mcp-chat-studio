Conclusion
MCP Chat Studio is a genuinely impressive tool that fills a real gap in MCP development tooling. The feature set rivals commercial API testing tools, and the dual Classic/Workspace modes show thoughtful UX design.

The main areas for improvement are:

Code organization (monolithic frontend)
Test coverage (especially frontend/E2E)
Security hardening (vm2, token management)
Scalability (in-memory storage â†’ database)
With these improvements, it could become the definitive MCP testing platform.

Verification Notes (code cross-check)
Correct
- vm2 is still used for NodeVM scripts; VM defaults to isolated-vm when available.
- Frontend is partially split: Assistant + Generator extracted; app.js remains large.
- No TypeScript (no tsconfig or TS sources).
- OAuth tokens are in-memory (server/services/OAuthManager.js uses Map).
- No database layer (localStorage + JSON files only).
- CI build step is a 10s npm start smoke check (.github/workflows/ci.yml).

Partial / Needs nuance
- Input sanitization exists (allowlist sanitizer in public/app/modals.js + template escaping in advanced-features; not a full DOMPurify-class sanitizer).
- Auth per-request is partial (OpenAPI auth is mapped into tool metadata and used in generated proxy code, but no full per-request auth UX like Postman).
- Sandbox engine is now abstracted (SandboxEngine.js) with isolated-vm default when installed; vm2 remains default for NodeVM scripts.

Incorrect / Outdated
- "API keys in localStorage" is not true for LLM keys. LLM keys are saved server-side in data/llm-config.json via server/routes/llm.js. localStorage stores sessions, scenarios, prompts, etc.
- "Request History Panel missing" is incorrect. History exists and persists locally (public/app.js session manager).
